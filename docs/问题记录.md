# 问题记录

## 1. Photoshop 自动化不稳定

现象：

- 直接通过 AppleScript 调 Photoshop 打开文件时，出现过 `不能获得 application "Photoshop 2025" (-1728)`。
- 用户重新登录 Photoshop 后，Photoshop UI 可以正常使用。

处理方式：

- 改用 Photoshop 菜单 `文件 > 脚本 > 浏览...` 执行 JSX。
- 已通过该方式成功执行：
  - `scripts/list_psd_layers.jsx`
  - `scripts/make_original_psd_overlay.jsx`
  - `scripts/export_psd_right_preview.jsx`

后续建议：

- 如果要做稳定流水线，优先探索 Photoshop UXP/ExtendScript 的命令行可执行方式。
- 保留 UI 脚本路径作为人工辅助方案，不作为完全自动化依赖。

## 2. PSD 预览读取问题

现象：

- 用 Python PIL 直接读取新版 PSD 合成预览时，导出的右侧区域出现彩条。
- 这不是 Photoshop 中的真实效果，而是 PIL 对 PSD 兼容预览读取不可靠。

处理方式：

- 改用 Photoshop 自身执行 `export_psd_right_preview.jsx` 导出 PSD 右侧预览。
- Photoshop 导出的预览与原版 PNG 做像素比对，结果一致。

后续建议：

- PSD 视觉校验以 Photoshop 导出为准。
- 不再把 PIL 直接读取 PSD 作为可靠校验依据。

## 4. Word 图表数据抽取复杂

现象：

- Word 中图表不是普通图片，而是 DOCX 内部 chart XML。
- 本轮脚本需要从 `word/charts/chart*.xml` 读取系列数据。

处理方式：

- 当前脚本已能抽取本轮需要的国债收益率曲线和资金利率曲线数据。
- 图表绘制仍是简化手绘版本。

后续建议：

- 如果数据源长期来自 Excel，建议直接以 Excel 作为图表数据源。
- Word 只作为文字素材来源，不建议承担结构化数据源角色。

## 5. 固定坐标覆盖导致版式问题

现象：

- 当前长图中出现大空白和叠加。
- 用户已明确指出“长图做的不行，中间很多空，还有叠加”。

根因：

- 旧长图被当作底图复用。
- 新内容长度与旧内容不一致时，固定坐标无法自动调整布局。
- 缺少组件高度计算、碰撞检测和全图重排。

后续建议：

- 下一版应改为组件化生成：背景、标题条、卡片、正文块、图表块逐个绘制。
- 所有元素位置由前一个元素的实际底部计算，禁止依赖硬编码偏移。

## 6. GitHub 认证问题

现象：

- 当前执行 `gh auth status` 显示 LLJ-code1 的 GitHub CLI token 失效。
- 通过普通 `git push` 仍然可以推送，说明 Git 凭据可用，但 GitHub CLI 不能用于 PR、issue、release 等操作。

影响：

- 可能影响创建 PR 或通过 GitHub CLI 操作仓库。
- 当前不影响直接推送 `main` 分支。

建议：

- 如后续需要用 `gh` 创建 PR、issue 或 release，需要重新执行：

```bash
gh auth login -h github.com
```

或在 Git 凭据弹窗中重新授权。

## 7. 首次 Git 推送因大文件中断

现象：

- 首次执行 `git push -u origin main` 时出现 `RPC failed; HTTP 400` 和远端断开。
- 本项目首个提交包含两个 PSD 文件，约 60M 和 63M。

处理方式：

- 设置本仓库 Git HTTP 兼容参数后重新推送成功：

```bash
git config http.version HTTP/1.1
git config http.postBuffer 524288000
git push -u origin main
```

结果：

- 已成功推送到 `https://github.com/LLJ-code1/zhaishichangtu` 的 `main` 分支。
- GitHub 提示 PSD 文件超过推荐 50MB，建议后续使用 Git LFS。

后续建议：

- 继续保留 PSD 作为项目素材可以接受，但不建议每轮都提交新的大 PSD。
- 后续如果要频繁迭代 PSD，应安装并启用 Git LFS。
