# 待解决清单

## P0：当前长图不能直接验收

### 1. 首屏空白过多

现象：

- 首屏正文结束后到卡片底部有大片空白。
- 视觉上像旧模板高度没有跟随新内容收缩。

原因判断：

- 当前生成逻辑复用了旧版长图卡片高度。
- 脚本主要做固定坐标覆盖，没有根据段落高度动态计算卡片高度。

建议修复：

- 首屏卡片改为内容自适应高度。
- 图片、标题、正文之间保留固定间距，正文结束后只留 40-70px 底部安全边距。
- 三个版本的首段文案控制在接近字数，避免高度差过大。

### 2. 债市分析板块文字与图标叠加

现象：

- “资金面”和“市场风险偏好”图标区域与右侧文字区域发生重叠。
- 部分文字覆盖到图标或浅色背景块上，阅读不清晰。

原因判断：

- 图标、正文、图表区域均为固定坐标。
- 文字换行后高度变化，脚本没有重新推开下方元素。
- 左侧图标列和右侧正文列没有明确的栅格边界。

建议修复：

- 债市分析改成清晰的两段式或卡片式布局：
  - 资金面：图标 + 文字 + 小图表。
  - 市场风险偏好：图标 + 文字。
- 给图标列和文字列设定不可重叠的边界。
- 每个信息块按内容高度计算位置，下一个块从上一个块底部继续排。

### 3. 中间多处空白不合理

现象：

- 首屏、债市分析、后市展望等区域都有不必要留白。
- 卡片之间和卡片内部的空间比例不均衡。

原因判断：

- 本轮采用旧长图作为底图，再局部覆盖文字和图表。
- 旧模板本身给原内容预留了较大空间，新内容更短后没有同步收缩。

建议修复：

- 放弃“旧图上覆盖”的主流程，改为从背景、卡片、标题条、正文、图表重新绘制。
- 每个 section 输出实际高度，整体长图高度由内容累加产生。

### 4. 图表呈现还不够精致

现象：

- 图表为脚本重绘的简化版本，视觉接近但不够像正式设计稿。
- 横轴日期文字曾与图例重叠，本轮临时关闭了横轴日期标签。

原因判断：

- PIL 手绘图表能力有限。
- 当前图表布局没有足够的图例和轴标签安全区。

建议修复：

- 用 matplotlib/Excel 导出图表，再嵌入长图，保证图表质量。
- 或在 PIL 中重写图表组件，明确标题区、绘图区、图例区、轴标签区。

### 5. PSD 不是真正完全可编辑版本

现象：

- 当前新版 PSD 可以打开并看到正确的“原版”长图。
- 但新版内容是一个顶层可见长图层，底层仍保留原模板，不是每段文字和每张图表都已拆成可编辑 PSD 图层。

原因判断：

- 本轮目标先保证 Photoshop 可打开的 PSD 交付。
- Photoshop 脚本直接编辑复杂 PSD 文本层、图表层的稳定性还未建立。

建议修复：

- 若后续需要设计师继续精修，保留当前 overlay PSD 即可。
- 若后续需要模板化复用，需要重建 PSD 图层映射规则：标题、日期、首段、表格、图表、各正文块分别对应可编辑层。

## P1：项目工程化待补

### 1. 需要明确最终生产流程

待确定：

- 最终以 PNG 自动生成为主，还是以 PSD 模板自动更新为主。
- 图表由 Word/Excel 抽取后重绘，还是直接从 Excel 导出图片。
- 三版首段文案由人工给定，还是脚本根据类型生成初稿。

### 2. 需要建立验收标准

建议至少包含：

- 无明显空白。
- 无文字、图标、图表叠加。
- 所有日期、收益率、bp 变化与 Word/Excel 来源一致。
- 三版只有首段文案不同，主体内容保持一致。
- 长图宽度固定 1125px，最终高度按内容自适应。

### 3. 大文件管理

当前 PSD 单文件约 60M-63M，仍低于 GitHub 100M 限制，但不适合长期频繁提交。

建议：

- 后续引入 Git LFS 管理 `.psd`。
- 输出图和 PSD 分版本归档，避免每次迭代都重复提交大文件。
