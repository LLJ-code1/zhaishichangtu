# 内容拆解与 PSD 输入规则

## 目标

把 Word 内容先拆成稳定字段，再交给 Photoshop 脚本排版。PSD 层不直接理解 Word 段落，也不负责判断哪些内容属于哪个版本。

## 输入文件

用户整理的三版 Word 已归入本周目录：

```text
/Users/a123/Downloads/债市周观察/债市周观察/weeks/2026-05-25_2026-05-29/inputs/word/variants/
├── 债市周观察原版.docx
├── 债市周观察（固收+）.docx
└── 债市周观察（债市）.docx
```

三份 Word 的结构已经统一：

- 17 个有效段落。
- 1 个国债收益率表。
- 2 个图表。
- 只有顶部投资建议段不同。
- 表格和图表在三版之间保持一致。

## 拆解原则

### 1. 公共母版字段

以下字段三版共用：

- 标题：`金葵花债市周观察`
- 副标题：`收益率整体下行`
- 债市表现一句话
- 国债收益率表
- 国债收益率曲线图
- 市场风险偏好
- 资金面
- 资金利率图
- 后市展望
- 数据来源
- 风险提示

### 2. 版本变量字段

三版只允许顶部投资建议段不同：

- `原版`：综合市场复盘口径。
- `固收+`：强调流动性、股债偏强共振和固收+配置价值。
- `债市`：强调纯债、波动和债市多头趋势。

债市版首段在结构化输出中统一修正为：

```text
虽然短期或仍有波动，但仍有较多机构存在欠配压力，债市多头趋势或仍能延续。建议继续持有纯债产品，具体详情请见下文。
```

源 Word 暂不改动，修正在 JSON 输出层完成。

### 3. 固定策略字段

配置策略不从 Word 抽取，固定为：

```text
境内债券市场：
建议以底仓价值和标配为主。
1.波动容忍度较低的客户，建议主要配置稳健低波、短债等产品。
2.对于存在一定波动容忍度的客户，可尝试优选"固收+"产品进行适当搭配。
```

固定标红：

- `底仓价值和标配`
- `波动容忍度较低`
- `一定波动容忍度`

## 结构化输出

脚本：

```text
scripts/extract_word_psd_content.py
```

输出：

```text
weeks/2026-05-25_2026-05-29/work/psd_content/
├── 原版.json
├── 固收+.json
├── 债市.json
└── index.json
```

每个 JSON 包含：

- `meta`：版本、来源 Word、日期、周报区间。
- `layout`：长图宽度和自适应高度策略。
- `fields`：PSD 需要替换的纯文本字段。
- `rich_text`：按标红规则切分后的红黑片段。
- `assets`：收益率表和 Word 内图表引用。

机器可读 schema：

```text
schemas/psd_content_schema.json
schemas/red_rules.json
```

## 标红规则

Word 不强制人工标红。当前由 `schemas/red_rules.json` 自动判断。

优先标红：

- 核心结论：`收益率整体下行约2BP`、`国债收益率全期限下行`
- 资金变化：`净投放1490亿元`、`资金利率上行`
- 版本策略：`固收+策略的配置价值凸显`、`继续持有纯债产品`
- 债市逻辑：`短期或仍有波动`、`欠配压力`、`债市多头趋势或仍能延续`
- 配置建议：`底仓价值和标配`、`波动容忍度较低`、`一定波动容忍度`

后续如果需要更精确，可以让 Word 人工标红，再改为优先读取 Word run 颜色。

## PSD 侧使用原则

PSD 脚本后续应只读取结构化 JSON：

- 不靠 Word 段落序号直接排版。
- 不从 Photoshop 图层里临时推断内容。
- 不统一首段字数。
- 根据 `rich_text` 拆红黑文本层。
- 根据文本测量结果调整卡片高度和后续区块 y 坐标。

自适应高度的基本顺序：

```text
测量首屏正文高度
-> 调整首屏卡片高度
-> 下移债市表现区块
-> 测量债市分析和后市展望区块
-> 下移配置策略和风险提示
-> 裁切最终长图高度
```
